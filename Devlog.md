Week 1 - Initialized separate repositories for the front and backend, everything looks fine, design archtechture looks pretty polished looking to use material UI, passport for authentication, and express session to keep user logged in. front end will be react, and backend will use js with node.js.

week 2 - Front End is seemlessly done, I provided div squares for each day, there are lot of techniques that go into making calendars it seems I need to fetch Date() everytime month changes, padding days are different since you don't technically know when the first day of the month starts. I was able to fix it by using new Date(currentYear, currentMonth[1] + 1, 0).getDate(); since 0 of the new month will automatically give you the last of the previous month.

week 3 - Back end was honestly bit more easier than front end, having more experience designing back-end I've successfully configured login, logout, session management through express and passport, and api endpoints for CRUD operations. one problem I've encountered is that sometimes even when the session cookies are set to expire in a month, they will sometimes expire prematurely within 20 - 30 minutes. after looking it up on stack overflow I've learned that if the session cookies are managed in the memory, sometimes another data may replace the session cookies and they maybe lost and require another one to be sent from the backend. solution was to use mongo store where session cookies will be sent and saved at mongoDB instead of local memory which can ensure the authentication will last until logged out or until the cookie is expired.

week 4 - Deployment experience was pretty bad, everything worked on my machine locally but moment I deployed front and back end, cracks started to appear. authentication cookies were not being sent to browser so even if I was logged in, since no authentication token was recieved, I couldn't create, read, edit, or delete any of the pre-existing notes. I ended up Updated the backend CORS middleware to explicitly allow the frontend's Render URL as the origin and set credentials: true, and adjusted Cookie Flags and set secure: true (requires HTTPS) and adjusted sameSite: 'none' to allow cross-site tracking between the client and server domain

I honestly learned about local vs production parity where locally, things are so much more forgiving, security policies like CORS and sameSite cookie attributes need to be thought out before deployment for sure.

More time spent on architecture would have been nice as well since I didn't have proper error boundaries which ended up creating technical debt where I wasn't sure where the error or bugs were from when I was recieving error codes.
